# preproc_out — AEGIS-KG Preprocessed Knowledge (CORR-024 v8/v9)

Generated by `python -m scripts.preprocess build` from `methodology-00/PREPROCESSING/`
+ `diagrams/`. This directory is the **machine-readable layer** of the AEGIS-KG
methodology; every JSON is structured (no raw_md) and indexed for fast lookup.

## v9: raw_md policy

- **`raw_md` is DROPPED** from `regulation/{REG}/_root/` (25 files, ~1.7 MB saved)
  because the body is duplicated in `regulation/{REG}/aggregated/` for
  01_SO/02_SR, or purely narrative for 00_README/03_validation/04_deduction
  where frontmatter alone is enough for audit.
- **`raw_md` is KEPT** in `crossregulation/`, `ambiguity_analysis/`, `global/`
  templates, and `_templates/` — these are catch-all narrative files
  without a structured counterpart.

Total files with `raw_md`: **84** (was 109 in v8).

## Quick Navigation

| You want to... | Look in... |
|---|---|
| Find a CSF 2.0 subcategory by ID | `entities/csfs/_meta/_index.json` → `entities/csfs/{FUNC}/<ID>.json` |
| Find a CSF 2.0 subcategory by Function | `entities/csfs/{GV,ID,PR,DE,RS,RC}/` |
| Find a regulation article | `regulation/{REG}/articles/Art_NN.json` |
| Find a clause by ID | `index/entities.json` (id → path) |
| Find all SOs for a regulation | `regulation/{REG}/aggregated/01_SecurityObjectives.json` |
| Get the full CSF 2.0 reference (incl. withdrawn) | `global/NIST_CSF_2.0_subcategories.json` |
| See what the build produced | `meta/manifest.json` (every shard path + sha256) |
| Check if a build had errors | `meta/build_info.json` (`error_count` field) |
